Sudoku solver

Part I  (To make a sudoku template where the user can solve the sudoku. It checks for correctness.)
Part II (The program must solve the sudoku. Different steps have different colors.)

Part I
1. Make the layout of the sudoku. It should be acceptable.
2. The input is taken from the user and the question is locked, user will not be able to change the values.
3. When a user inputs a number, it is validated and feedback is given.

Part II
1. The program takes an input and then creates a possibility matrix.
2. It checks for the easy cases, the low hanging fruit.
3. It checks for 2,3,4 values and eliminates them from the possibilities.
4. A color is given for every step that the sudoku solves, so the user knows the order the numbers were solved.
